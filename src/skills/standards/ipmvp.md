# IPMVP — International Performance Measurement & Verification Protocol

## Quick facts
- **Owner**: EVO (Efficiency Valuation Organization), non-profit
- **Why it exists**: standardize "how much energy did this retrofit actually save"
- **When to use**: any project where savings are claimed (ESCO, EPC, T-VER, ISO 50001 reporting)
- Latest version: 2022 (formerly Volume I 2012)

## The 4 M&V Options

| Option | What it measures | When to use |
|---|---|---|
| **A** | Retrofit-isolation, **key parameter** | Single device, easy to isolate, some parameters estimated (e.g. lighting hours) |
| **B** | Retrofit-isolation, **all parameters** | Single device, all relevant parameters measured (e.g. motor + VFD on one fan) |
| **C** | **Whole facility** | Building/plant-level meter readings, regression on baseline drivers |
| **D** | **Calibrated simulation** | New construction or major retrofit where baseline doesn't exist |

## Choosing options
- **Option A** = cheapest, lowest accuracy — good for low-stakes retrofits
- **Option B** = direct measurement, medium cost, high accuracy
- **Option C** = whole-building, accounts for all interactions, needs ≥12 months baseline
- **Option D** = simulation, only when no baseline available (greenfield)

## Key concepts
- **Baseline period** — typically 12 months pre-retrofit (covers seasonal variations)
- **Reporting period** — duration of guaranteed/measured savings (often 1–5 years)
- **Adjustments**:
  - **Routine** — weather, occupancy, production (independent variables)
  - **Non-routine** — equipment changes, expansion (must be agreed in M&V plan)
- **Boundary** — what's inside vs outside the project (critical for Option B)

## M&V Plan must include
1. Project scope + boundary
2. Option chosen + justification
3. Baseline data + dates
4. Measurement equipment + accuracy
5. Reporting period + frequency
6. Adjustment formulas
7. Responsibilities + reporting format

## Common pitfalls
- Picking Option C when production changes wildly (regression breaks)
- Baseline period too short → seasonal bias
- Equipment accuracy not documented → disputed savings
- Non-routine adjustments not pre-agreed → fights at year-end
- Forgetting that "savings" = baseline-adjusted minus reporting period (not raw difference)

## Standards aligned with IPMVP
- **ISO 50015** — General principles for M&V
- **ASHRAE Guideline 14** — measurement of energy + demand savings
- **FEMP** M&V Guidelines (US Federal)
- T-VER project methodology (Thailand) typically references IPMVP Option B/C
