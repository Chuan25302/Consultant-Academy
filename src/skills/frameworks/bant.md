# BANT — Skill Card

**Budget · Authority · Need · Timeline** — qualify a deal in 4 questions.

## When to use
- Early-stage qualification, before investing time on proposal
- Distinguish "interested" from "buying"
- Especially useful when consultant pipeline > capacity

## The 4 questions (in Thai)

| Letter | คำถามที่ใช้จริง |
|---|---|
| **B**udget | "งบที่ฝ่าย procurement กันไว้สำหรับ project แบบนี้ประมาณเท่าไหร่ครับ?" |
| **A**uthority | "ใครเป็นคน final sign-off เรื่องงบประมาณนี้ครับ — ผู้จัดการโรงงาน หรือ ขึ้นไปถึง board?" |
| **N**eed | "อะไรเป็นปัจจัยที่ทำให้พิจารณาทำตอนนี้ ไม่ใช่ปีหน้า?" |
| **T**imeline | "ถ้าทุกอย่าง smooth — เริ่ม implement ได้ในไตรมาสไหน?" |

## How to interpret
- **B unclear**: budget เป็น signal ใหญ่สุด — ถ้าตอบไม่ได้ = ยังไม่จริง
- **A wrong**: คุยอยู่กับคนที่ไม่ใช่ decision-maker = ต้องหา champion ที่พาขึ้น
- **N weak**: ถ้าไม่มี trigger ชัด → จะ delay ไปเรื่อยๆ
- **T vague**: "ดูไปก่อน" = no timeline = no project

## Common Thai pitfalls
- ลูกค้าตอบ B = "เท่าที่จำเป็น" — ต้อง probe ต่อ "หมายถึงราคาประมาณเท่าไหร่ครับ"
- A: คนที่ดูเหมือนใหญ่อาจไม่ใช่ — ต้องเช็ค finance approval limit
- ในวัฒนธรรมไทย questioning timeline ตรงๆ อาจเป็นเรื่องเสียหน้า — ใช้ soft phrasing
